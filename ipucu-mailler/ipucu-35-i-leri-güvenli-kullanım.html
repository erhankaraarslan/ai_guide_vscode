İpucu 1

Hassas veri yapıştırmayın

Şifre, API key, token, PII (kişisel bilgi) AI sohbetine göndermeyin. Responsible use: VS Code hassas bilgi taraması yapar. Kod örneği verecekseniz placeholder kullanın. Trust Center'da veri işleme politikalarını inceleyin.

Nasıl yapılır:

  1. Gerçek değer yerine PLACEHOLDER
  2. API key → process.env.API_KEY
  3. Şifre → '***' veya env ref

Deneyebileceğiniz örnek:

apiKey: 'sk-...' yerine apiKey: process.env.API_KEY

Kaynak: GitHub Docs
İpucu 2

Üretilen kodu güvenlik taraması

AI kodu SQL injection, XSS, path traversal açığı içerebilir. Özellikle user input kullanan kodda manuel veya otomatik tarama yapın. Responsible use: önerilen kodu her zaman doğrulayın; AI hataya açıktır.

Nasıl yapılır:

  1. User input → sanitize/validate
  2. SQL → parametreli sorgu
  3. AI'dan gelen kodu güvenlik gözüyle inceleyin

Deneyebileceğiniz örnek:

req.body.email direkt SQL'e gitmemeli

Kaynak: GitHub Docs
İpucu 3

Kurumsal ayarlar

GitHub Copilot Business/Enterprise ile veri eğitim politikaları, filtreleme, audit log. Business planında kodunuz eğitime gitmez. Şirket politikasına uyun; IP, uyumluluk soruları için Trust Center.

Nasıl yapılır:

  1. Yönetici ayarlarını kontrol edin
  2. Eğitimden hariç tutma (opt-out)
  3. SSO, IP kısıtlaması varsa uygulayın

Deneyebileceğiniz örnek:

Şirket verisi eğitime gitmemeli → ayarlardan kapat

Kaynak: GitHub Trust Center
