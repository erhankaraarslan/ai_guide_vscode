Prompt dosyaları

İpucu 1

.github/prompts/*.prompt.md

Sık yaptığınız görevler için .prompt.md oluşturun. /komut-adı ile çağırırsınız; önceden tanımlı istem, ajan ve model yüklenir. Kod incelemesi, güvenlik taraması, refaktör için ayrı prompt dosyaları.

Nasıl yapılır:

  1. .github/prompts/ oluşturun
  2. code-review.prompt.md gibi dosya
  3. İçine prompt: ve agent: tanımlayın

Deneyebileceğiniz örnek:

/code-review yazınca inceleme ajana geçer

Kaynak: VS Code Context Guide
İpucu 2

/code-review örneği

Kod incelemesi için: agent: inceleme ajanı, prompt: standart sorular. Cookbook'daki güvenlik, performans, okunabilirlik sorularını özelleştirin. /code-review yazınca hepsi hazır gelir.

Nasıl yapılır:

  1. code-review.prompt.md oluşturun
  2. prompt: «Güvenlik, performans, okunabilirlik incele»
  3. agent: inceleme ajanı adı

Deneyebileceğiniz örnek:

prompt: «Bu değişiklikleri incele: güvenlik, performans»

Kaynak: GitHub Docs
İpucu 3

/prompts ile yönetin

Chat'te /prompts yazarak mevcut prompt dosyalarını listeleyebilir, yenisini ekleyebilirsiniz. /rename ile sohbeti adlandırın, /delete ile silin. Slash komutları iş akışını hızlandırır.

Nasıl yapılır:

  1. Chat'e /prompts yazın
  2. Mevcut prompt'ları görün
  3. Configure Prompts ile ekleyin/düzenleyin

Deneyebileceğiniz örnek:

/prompts → code-review, security-check ekleyin

Kaynak: GitHub Docs
