İpucu 1

Çalışma alanı indekslemesi

VS Code projeyi indeksler; #codebase ile hızlı semantik arama. Basic (daha basit algoritma), local (yerel semantik) veya remote (GitHub depo) indeks seçenekleri. Büyük depolarda remote daha hızlı.

Nasıl yapılır:

  1. Proje açıkken arka planda indekslenir
  2. Status bar'da durumu kontrol edin
  3. Büyük depo için remote index açın

Deneyebileceğiniz örnek:

1000+ dosyada #codebase ile 'auth nerede?'

Kaynak: VS Code Docs
İpucu 2

Proje düzeyinde talimatlar

Mimari, modül sınırları, API sözleşmeleri gibi AI'ın koddan çıkaramayacağı bilgileri copilot-instructions.md'de verin. PRODUCT.md (ürün vizyonu), ARCHITECTURE.md (mimari), CONTRIBUTING.md (katkı rehberi) ile karar verme bağlamını zenginleştirin.

Nasıl yapılır:

  1. ARCHITECTURE.md, CONTRIBUTING.md referansları
  2. Modül sınırları: 'X Y'yi import etmemeli'
  3. Kısa ve odaklı

Deneyebileceğiniz örnek:

«API layer sadece services/ ile konuşur»

Kaynak: VS Code Context Guide
İpucu 3

Monorepo'da odaklı bağlam

Multi-root workspace'te AI'a hangi köke odaklanacağını net verin. #packages/auth gibi spesifik referans. Bağlam mühendisliği ölçeklendirmede proje geneli, modül ve özellik katmanları düşünün.

Nasıl yapılır:

  1. File > Add Folder ile multi-root
  2. #packages/api gibi belirli kök
  3. Tüm monorepo'yu eklemeyin

Deneyebileceğiniz örnek:

#packages/api veya #apps/web

Kaynak: VS Code Context Guide
