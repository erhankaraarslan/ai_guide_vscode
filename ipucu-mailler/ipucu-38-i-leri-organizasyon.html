İleri · Organizasyon

Ekip için standartlar

İpucu 1

copilot-instructions.md ekip sözleşmesi

Tüm ekip aynı talimatlara uyar. Kod stili, commit mesajı formatı, PR şablonu, test kuralları. Organizasyon seviyesinde accessibility (WCAG) gibi standartlar da tanımlanabilir. Tek kaynak.

Nasıl yapılır:

  1. .github/copilot-instructions.md'de ekip kuralları
  2. CONTRIBUTING.md referansı
  3. Yeni üye /init + düzenleme

Deneyebileceğiniz örnek:

## Commit: feat/fix/refactor prefix. ## Test: her public fn test.

Kaynak: GitHub Accessibility
İpucu 2

Ortak Skill deposu

Şirket veya ekip için skill'leri ortak depoda tutun. Her proje .github/skills/'e submodule veya script ile çeker.

Nasıl yapılır:

  1. Ortak repo: org/ai-copilot-skills
  2. Projede git submodule add
  3. Veya setup script ile kopyala

Deneyebileceğiniz örnek:

git submodule add …/ai-skills .github/skills/shared
İpucu 3

Prompt şablonları

code-review, security-check, release-notes gibi standart prompt'ları paylaşın. Yeni projede aynı .prompt.md'ler kullanılır.

Nasıl yapılır:

  1. .github/prompts/ şablonları repo'da
  2. Template repo ile yeni proje
  3. Kurulumda prompts/ kopyalanır

Deneyebileceğiniz örnek:

Tüm projelerde /code-review aynı sorularla
