Plan ajanı ile ilk plan

İpucu 1

Plan ajanı ne yapar?

Büyük görevi adım adım uygulama planına çevirir. Kod yazmadan önce strateji oluşturur; plan-template.md ile tutarlı format. Belirsiz noktaları size sorar. GitHub issue referansı ile bağlam ekleyebilirsiniz.

Nasıl yapılır:

  1. Mod seçiciden 'Plan' seçin
  2. Görevinizi yazın (örn. 'Kullanıcı kayıt akışı')
  3. AI sorular sorar, siz cevaplayın, plan oluşur

Deneyebileceğiniz örnek:

«Sepet özelliği: ürün ekleme, miktar, silme, toplam»

Kaynak: VS Code Context Guide
İpucu 2

Planı inceleyip onaylayın

Plan oluşunca baştan sona okuyun. Eksik/yanlış varsa düzeltme isteyin. Doğru görünüyorsa 'Start Implementation' ile Agent'a geçin.

Nasıl yapılır:

  1. Planı okuyun
  2. Eksikse «2. adımda migration detayı ekle» deyin
  3. Memnunsanız handoff düğmesine basın

Deneyebileceğiniz örnek:

Plan 5 adım gösteriyorsa Agent'a geçince bu 5 adım bağlamda olur
İpucu 3

Büyük özelliği parçalayın

Tek mesajda 'tüm özelliği yap' demek yerine adım adım isteyin. Karmaşık görevleri küçük, yönetilebilir parçalara bölün. Her adımı onaylayıp sonrakine geçin.

Nasıl yapılır:

  1. 1) «Önce veritabanı şemasını tasarla»
  2. Onayla
  3. 2) «API endpoint'lerini yaz»
  4. Onayla
  5. 3) «Frontend ekle»

Deneyebileceğiniz örnek:

Şema → onayla → API → onayla → Frontend

Kaynak: GitHub Docs
