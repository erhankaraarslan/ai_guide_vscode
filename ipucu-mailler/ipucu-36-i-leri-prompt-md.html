Prompt dosyası ileri düzey

İpucu 1

agent: + prompt: + model:

.prompt.md'de agent: hangi ajana geçileceği, prompt: varsayılan istem, model: sabit model. Tek komutla tam kurulum.

Nasıl yapılır:

  1. agent: inceleme-ajani
  2. prompt: «Bu değişiklikleri incele...»
  3. model: ['Claude Sonnet']

Deneyebileceğiniz örnek:

/code-review → inceleme ajana geç, prompt hazır, model sabit
İpucu 2

Placeholder ve değişkenler

VS Code prompt dosyalarında ${fileBasename}, ${relativeFile} gibi placeholder'lar desteklenir. Örn. ${fileBasename} ile seçili dosya adı enjekte edilir. Dokümantasyonda tam listeyi kontrol edin.

Nasıl yapılır:

  1. Placeholder varsa kullanın
  2. Örn: ${fileBasename}, ${relativeFile}
  3. Desteklenmiyorsa sabit metin

Deneyebileceğiniz örnek:

prompt: «${fileBasename} dosyasını incele»
İpucu 3

Prompt kategorileri

Farklı iş türleri için: /code-review, /security-check, /refactor, /explain. Her biri ayrı .prompt.md. /prompts ile hepsini yönetin.

Nasıl yapılır:

  1. .github/prompts/ altında kategorize edin
  2. security-check.prompt.md, refactor.prompt.md
  3. Kısa isimler: /sec, /refactor

Deneyebileceğiniz örnek:

/sec → güvenlik. /refactor → refaktör.
