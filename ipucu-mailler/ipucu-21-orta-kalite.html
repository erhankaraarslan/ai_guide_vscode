AI çıktısını doğrulama

İpucu 1

Kabul etmeden önce inceleyin

AI ürettiği kodu kabul etmeden önce okuyun. Fonksiyonellik, güvenlik, okunabilirlik ve sürdürülebilirliği düşünün. Gerekirse Copilot Chat'ten kodu açıklamasını isteyin. Kenar durumları, null kontrolü, hata mesajlarını kontrol edin.

Nasıl yapılır:

  1. Önerilen değişikliği diff olarak görün
  2. Her satırı gözden geçirin
  3. Şüphe varsa reddedin

Deneyebileceğiniz örnek:

Boş liste, null, çok uzun string — bu girdilerde ne oluyor?

Kaynak: GitHub Docs
İpucu 2

Test çalıştırın

AI değişiklik yaptıktan sonra ilgili testleri çalıştırın. Linting, code scanning, IP scanning gibi araçlarla ek doğrulama katmanı ekleyin. AI 'test et' demiş olsanız bile sonucu siz kontrol edin.

Nasıl yapılır:

  1. npm test veya proje test komutu
  2. AI çalıştırmışsa çıktıyı kontrol edin
  3. Test yoksa manuel senaryo deneyin
  4. Linting ve tarama araçları kullanın

Deneyebileceğiniz örnek:

Kod değişikliği sonrası: npm test

Kaynak: GitHub Docs
İpucu 3

Güvenlik kontrolü

Enjeksiyon, hardcoded API key, eksik input validasyonu. Responsible use: şifre, token, PII sohbetine göndermeyin. Trust Center'da gizlilik ve IP politikalarını inceleyin.

Nasıl yapılır:

  1. Üretilen kodu güvenlik açısından tarayın
  2. Env variable, secret manager kullanılmalı
  3. User input için validasyon kontrol edin

Deneyebileceğiniz örnek:

API key kodda görünüyorsa → env variable

Kaynak: GitHub Docs
